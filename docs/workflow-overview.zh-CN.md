# Write Agent 内容生产总流程图（单总图）

本图用于面试讲解与团队沟通，聚焦内容生产主链：
- 可选热点入口（GitHub/Linux.do）
- 改写与主编审核闭环
- 封面生成（含目标文章与手动输入分支）
- 排版输出

## Mermaid（可维护版本）

```mermaid
flowchart LR
  %% 入口
  G[GitHub 热点] --> H[热点选题]
  X[Linux.do 热点] --> H
  H --> I[选题与输入]
  M[手动输入原文] --> I

  %% 改写任务
  I --> J[启动改写任务]
  J --> CP[[支持中断恢复\n避免重复副作用]]
  J --> SSE[实时进度回传]
  SSE --> U[页面进度一致展示]

  %% 审核闭环
  J --> R[主编审核]
  R -->|通过| SRC{封面来源}
  R -->|未通过| RETRY[进入重试改写\n最多二轮闭环]
  RETRY --> J

  %% 封面分支
  SRC -->|目标文章| C[生成封面]
  SRC -->|手动标题+正文| MC[创建手动封面上下文]
  MC --> C

  %% 输出
  C --> L[排版输出]
  L --> O[最终成稿]

  %% 失败终态
  R -->|异常失败| F[失败终态]
  F --> T[[可用 trace_id\n快速排障]]
```

## 讲解口径（3分钟）

1. 入口可来自热点，也可直接手动输入原文。
2. 核心执行是“改写任务 + 主编审核”的闭环，未通过会进入重试，最多二轮。
3. 任务启动阶段支持中断恢复，并通过幂等机制避免重复副作用。
4. 前端通过实时进度回传保持状态一致。
5. 审核通过后进入封面生成，既支持目标文章，也支持手动标题+正文。
6. 若失败进入终态，可用 `trace_id` 快速定位问题。
