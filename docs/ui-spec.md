# 写作智能体 UI 重设计说明（给 Gemini 3 Pro / 外部设计方）

## 1. 文档目的与使用方式

### 1.1 目的
本说明用于指导外部设计方或 Gemini 3 Pro 产出写作智能体的完整 UI 重设计方案。
要求输出不是泛化页面，而是可落地到真实页面结构、组件与交互状态的设计稿。

### 1.2 使用方式
1. 先阅读本说明全文，理解产品范围与页面职责。
2. 按“页面级设计说明”逐页生成 UI 方案（Desktop + Mobile）。
3. 设计结果必须覆盖：布局、组件、交互、状态、异常场景、响应式。
4. 不得改动业务流程，不引入新的后端字段或 API 前提。

### 1.3 设计边界
- 仅做 UI/交互重设计，不改后端接口契约。
- 保持现有信息架构（5 个页面 + 左侧导航）。
- 允许在视觉层与信息层级上做明显优化，但业务功能保持一致。

---

## 2. 产品范围与页面地图（信息架构）

### 2.1 页面地图
- 写作改写：`/`
- 风格管理：`/styles`
- 素材库：`/materials`
- 审核：`/reviews`
- 封面：`/covers`

### 2.2 全局导航
- 左侧 Sidebar 固定导航（可折叠）
- 顶层品牌区：写作智能体
- 主内容区：每个路由对应独立页面

### 2.3 当前业务主流程
风格提取 ->（可选）素材管理/RAG -> 改写（流式）-> 审核 -> 封面生成

---

## 3. 全局 UI 设计原则（基于现有 Demo 调性）

1. 专业克制：避免花哨炫技，强调“生产工具感”。
2. 信息优先：输入区、结果区、状态反馈要清晰分层。
3. 流程导向：让用户始终知道“现在在第几步、下一步做什么”。
4. 可读性优先：长文本场景要有舒适行宽、层级清晰的排版。
5. 低认知切换：跨页按钮、状态色、反馈样式尽量统一。
6. 响应明确：任何耗时操作必须有过程反馈（尤其 SSE 流式任务）。

---

## 4. 全局规范

### 4.1 布局网格（桌面/平板/移动）
- Desktop（>=1280）：Sidebar + 双栏主布局优先。
- Tablet（768-1279）：Sidebar 可折叠，内容区可改为上下布局。
- Mobile（<768）：单栏流式布局，操作按钮固定底部或就近吸附。

### 4.2 间距与层级
- 页面层级：Page Header -> Primary Content -> Secondary Content。
- 卡片内边距与组件间距统一节奏（例如 8/12/16/24）。
- 重要操作按钮与次要按钮在视觉权重上明显区分。

### 4.3 色彩与对比度
- 主色用于关键动作（提交、生成、确认）。
- 成功/失败/进行中必须使用固定语义色。
- 暗色主题下保持足够可读对比度（正文、次级文本、边框）。

### 4.4 字体层级
- H1：页面标题
- H2/H3：区块标题
- Body：正文/输入内容
- Meta：时间、状态、辅助描述

### 4.5 动效与反馈节奏
- 页面进入：轻量过渡，不影响效率。
- 列表加载：骨架屏或占位状态。
- 流式输出：使用持续可见的“进行中”反馈，不闪烁。
- 弹窗：进入/退出统一时长与缓动。

---


## 6. 页面级设计说明（5 页）

## 6.1 写作改写页（`/`）

### 页面目标
让用户完成“输入原文 -> 选风格 -> 设目标字数 -> 流式获取改写结果”。

### 页面布局分区
- 左区（输入控制区）：
  - 原文输入框
  - 风格选择
  - 目标字数选择
  - 开始改写 / 取消
- 右区（结果区）：
  - 改写结果（流式）
  - 字数统计
  - 复制按钮
  - 历史记录入口与详情

### 关键组件（必有）
- Textarea：原文输入
- Select：风格、目标字数
- Button：开始改写、取消、复制
- Result Panel：支持流式追加显示
- History List + Detail Modal

### 主流程与交互
1. 用户输入原文并选择风格。
2. 点击“开始改写”，进入流式过程。
3. 结果区逐步追加内容，同时实时更新字数。
4. 完成后可复制，且历史记录刷新。

### 状态设计
- 初始：空输入提示
- 加载中：按钮 loading、结果区显示进行中
- 空态：无历史记录提示
- 错误态：提示改写失败原因
- 成功态：显示完整结果与字数
- 流式中：结果持续更新、允许取消

### 异常与边界
- 未选风格/无输入时禁用开始按钮。
- 用户取消后停止流式连接并保留已生成内容。
- 长文本输入时避免页面卡顿（可分块渲染）。

### 移动端适配
- 改为上下结构：输入区在上、结果区在下。
- 主要动作按钮保持可见（吸底或就近）。

---

## 6.2 风格管理页（`/styles`）

### 页面目标
管理已有风格，并通过多篇参考文章提取新风格（支持流式进度与预览）。

### 页面布局分区
- 风格列表区（左或上）
- 风格详情区（右或下）
- 新建风格弹窗

### 关键组件（必有）
- Style List：名称 + 简述 + 删除
- Style Detail：12 维风格字段折叠/展开
- Create Modal：
  - 风格名称输入
  - 多参考文章输入（可增删）
  - 流式状态文案
  - 增量内容预览

### 主流程与交互
1. 选择已有风格查看详情。
2. 点击“新建”打开弹窗。
3. 输入名称 + 多篇文章并提交。
4. 显示流式进度和预览，完成后自动刷新列表。

### 状态设计
- 初始：无风格时引导创建
- 加载中：风格列表 skeleton
- 空态：无风格提示
- 错误态：提取失败提示 + 保留输入
- 成功态：新风格入库并可查看
- 流式中：状态消息 + 预览文本实时刷新

### 异常与边界
- 文章输入全空时禁用提交。
- 至少保留一个文章输入框。
- 删除风格需二次确认。

### 移动端适配
- 列表与详情改为折叠分区或 Tab 切换。
- 弹窗改全屏抽屉，输入区可滚动。

---

## 6.3 素材库页（`/materials`）

### 页面目标
维护 RAG 素材（新增、查看摘要、删除），用于改写时检索增强（可选功能）。

### 页面布局分区
- 顶部工具栏（添加素材）
- 素材列表（卡片）
- 新增素材弹窗

### 关键组件（必有）
- Material Card：
  - 标题
  - 内容摘要
  - 标签
  - 来源
  - 时间
  - 删除按钮
- Add Material Modal：
  - 内容（必填）
  - 来源（可选）
  - 标签（可选）

### 主流程与交互
1. 用户点击“添加素材”。
2. 在弹窗填写内容并提交。
3. 提交后关闭弹窗并刷新列表。
4. 可删除素材，删除后列表即时更新。

### 状态设计
- 初始：空态引导
- 加载中：列表占位
- 错误态：新增/删除失败提示
- 成功态：操作成功反馈
- 空态：暂无素材 + CTA

### 异常与边界
- 内容为空时禁用提交。
- 删除操作必须二次确认。
- 标签显示需处理长文本截断。

### 移动端适配
- 卡片改单列，操作按钮右上固定。
- 标签换行策略要明确，避免撑爆布局。

---

## 6.4 审核页（`/reviews`）

### 页面目标
浏览改写记录的审核状态，并快速查看原文/改写结果详情。

### 页面布局分区
- 列表区（记录卡片）
- 详情弹窗（点击“查看详情”）

### 关键组件（必有）
- Review Record Card：
  - 记录 ID
  - 风格名
  - 状态（完成/失败/处理中）
  - 创建时间
  - 原文摘要
  - 结果摘要
  - 查看详情按钮
- Detail Modal：
  - 原文全文
  - 改写全文
  - 错误信息（如果有）

### 主流程与交互
1. 加载改写记录列表。
2. 用户查看摘要并筛选重点记录（可选）。
3. 点击详情查看完整内容。

### 状态设计
- 加载中：列表 loading
- 空态：暂无审核记录
- 错误态：列表加载失败提示
- 成功态：可浏览与查看详情

### 异常与边界
- 失败记录需突出错误状态。
- 无改写结果时显示占位文本而非空白。

### 移动端适配
- 卡片纵向堆叠，摘要长度控制。
- 详情弹窗在移动端改全屏。

---

## 6.5 封面页（`/covers`）

### 页面目标
基于已完成改写生成封面图，并管理封面风格与历史。

### 页面布局分区
- 左：参数控制区
  - 文章选择
  - 生成方式（自动/风格/自定义）
  - 风格选择或自定义 Prompt
  - 比例选择
  - 生成按钮
- 右：预览区
  - 当前封面预览
  - 下载、重新生成
- 下方：风格管理区 + 封面历史区

### 关键组件（必有）
- Rewrite Select（仅展示已完成记录）
- Prompt Mode Segmented Control
- Ratio Select（2.35:1 / 1:1 / 9:16 / 3:4）
- Preview Panel（图片 + 操作）
- Cover Style List（删除）
- Create Cover Style Modal（名称/模板/描述）
- Cover History Grid

### 主流程与交互
1. 选择改写记录。
2. 选择生成模式并配置参数。
3. 点击生成，展示过程状态。
4. 生成完成后在预览区查看并下载。
5. 通过风格管理维护可复用模板。

### 状态设计
- 初始：未选文章提示
- 加载中：生成按钮 loading + 过程提示
- 空态：无封面历史/无风格时引导创建
- 错误态：生成失败提示
- 成功态：展示图片、下载可用
- 流式中：状态逐步更新（start/prompt/generating/done）

### 异常与边界
- 未选择文章时禁用生成按钮。
- 风格模式但未选风格时给出明确提示。
- 网络失败时保留当前输入参数，方便重试。

### 移动端适配
- 参数区与预览区改为上下布局。
- 历史区改横向滚动卡片或两列瀑布流（保持可点击）。

---

## 7. 文案与微交互规范

### 7.1 文案规范
- 按钮文案用动词开头：生成封面、开始改写、提取风格、添加素材。
- 错误提示说明“原因 + 下一步建议”。
- 空态文案给出下一步入口，不只写“暂无数据”。

### 7.2 反馈规范
- 成功反馈：短提示 + 结果可见区域高亮。
- 失败反馈：就近显示（表单内/结果区），避免用户找不到错误点。
- 危险操作：删除必须二次确认。

### 7.3 可访问性
- 关键按钮、输入框需有清晰 focus 态。
- 图标按钮必须有 tooltip/aria-label。
- 颜色不能作为唯一状态信息来源（需配文案/图标）。

---

## 8. 交付验收清单（给外部设计方）

1. 是否覆盖 5 个页面及其核心流程。
2. 每页是否包含：目标、布局、组件、交互、状态、移动端说明。
3. 是否保留既有业务前提（无新增后端字段要求）。
4. 是否统一跨页面组件与语义反馈。
5. 是否提供至少 Desktop + Mobile 两套关键页面示意。
6. 是否对流式任务给出明确反馈方案。
